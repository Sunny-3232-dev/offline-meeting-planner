# オフ会プランナー 🎉

はじめてのオフ会主催を、企画から告知まで背中を押すツール。

「オフ会を開いてみたいけど、何から始めればいいかわからない」——そんなリベシティ会員のために、
プロフィールを起点に **企画 → 基本情報 → 進行イメージ → 詳細情報 → 画像 → チャット作成 → 告知** までを一気通貫でサポートします。

[Skill Market Creator Pro](https://github.com/Sunny-3232-dev/Skill-Market-Creater-V4)・
[ノウハウ図書館 編集部](https://github.com/Sunny-3232-dev/Google-AI-Studio-KnowHowLib) に続く「背中を押すツール」第3弾です。

## 使い方（8ステップ）

| ステップ | 内容 |
|---|---|
| 1. プロフィール | 自己紹介・興味・**開催したい場所（対面／オンラインの必須選択、対面時は開催エリア）**・既に決まっている企画テーマ（任意）・初主催の不安を入力 |
| 2. 企画案 | **お金の5つのテーマ（貯める・稼ぐ・守る・増やす・使う）とその他で各4案ずつ提案**。プロフィールでテーマを指定した場合はそのテーマに沿った案を5件だけ生成。各案にペルソナ（来てほしい人）つき。ピン留め（灯マーク）は**最大4件**、「別の案を出す」で再生成してもピン留めは残る。カードを**ダブルクリック**するとその企画で次へ直行 |
| 3. 基本情報 | タイトル（AI候補5件）、日時・場所（必須入力）、定員（AIが目安を提案）。オフライン時は**リベシティ公式オフィス（14拠点）をラジオボタンで選択**でき、場所欄・詳細文・サムネプロンプトに連動（オフィス予約の注意つき）。オンライン時は**oVice／Zoom／Google Meet／Teams／その他**から開催ツールを選択。完了時に画像プロンプト2種を裏で先行生成 |
| 4. 進行イメージ | 人数×時間からAIが生成。行の追加・削除・並べ替え・時間変更が自由（時刻は自動再計算） |
| 5. 詳細（公開情報） | オフ会チャット作成フォームの「詳細（公開情報）」欄にそのまま貼れる文章を、**■イベント・オフ会内容／■参加費用／■参加方法／■募集期限／■注意事項／▶オフ会ガイドライン**の固定テンプレート構成で生成。**書き直しヒントのチップ**（カジュアルに／絵文字増減／オラオラ系／関西弁など）をクリックすると要望欄に入り、その内容でAIが書き直す。**マークダウン記法は使わず**プレーンテキストで出力（リベシティが非対応のため）。**チャット作成フォーム用のタグも同時生成**しコピーできる |
| 6. 画像 | チャットアイコン＆告知サムネイルの**画像生成AI用プロンプト**（「あなたはプロのデザイナーです。」始まり）を縦並びで表示。ChatGPT / Gemini 起動リンクつき。アイコンはAIが**キーワードのみ**生成しプロンプト文はコード側で組み立て、サムネイルは**キャッチーなタイトル・日時・場所を画像に焼き込む**プロンプトを生成 |
| 7. チャット作成 | リベシティのオフ会チャット作成フォームへ転記する**ルーム名・定員・タグ・詳細（公開）情報・限定公開情報**をカードごとにコピーできる |
| 8. 告知 | **オフ会チャットURLを入力**した上で、支部チャット向け・つぶやき向けの文章を生成。**公式支部チャット8つ（北海道／東北／関東／北陸・信越／東海／近畿／中国・四国／九州・沖縄）から投稿先を選ぶと「〇〇支部チャットを開く」の直リンク**を表示（プロフィールの開催エリアから自動推定）。オフ会チャットURLが本文に結合され、「この内容でつぶやく」ボタンで本文入力済みのつぶやき画面が開く |

## 複数オフ会の記録

作ったオフ会は**最大4件**までトップ画面に保存され、「開く」でいつでも編集を再開できます。
オフ会が終了したらゴミ箱ボタンで削除してください（4件埋まっていると新しいオフ会は企画できません）。

## 前工程を変更したときの「作り直しますか？」確認

前の工程の内容を変更してから次に進むと、下流の生成物が前提と食い違っていないか自動チェックし、
食い違っていれば「作り直しますか？」と確認してから作り直します。

- 企画案を選び直す（既存オフ会の企画を変更） → 進行イメージ・詳細情報・画像・告知文をまとめて作り直すか確認
- 基本情報を変更 → 進行イメージ・画像を作り直すか確認
- 進行イメージを変更 → 詳細（公開情報）を作り直すか確認
- 詳細（公開情報）を変更 → 展開用の文章（支部チャット・つぶやき）を作り直すか確認

「いいえ」を選んだ場合は古い内容のまま次に進み、以後は同じ変更点について再度は聞きません（さらに変更が入るまで）。

## 技術構成

- React 19 + Vite + TypeScript + Tailwind CSS 4
- Google Gemini API（`@google/genai` / gemini-2.5-flash）
- ホスティング: Google AI Studio（APIキーはセッションから自動取得）
- データ保存: ブラウザのlocalStorageのみ（サーバー送信なし）

## 開発

```bash
npm install
# .env.local に GEMINI_API_KEY=... を設定（ローカル開発用）
npm run dev
```

- 型チェック: `npm run lint`
- ビルド: `npm run build`

## APIキーの解決順序

1. `window.aistudio.apiKey`（Google AI Studioセッション）
2. 画面右上の鍵アイコンから手動保存したキー（localStorage）
3. 環境変数 `GEMINI_API_KEY` / `API_KEY`（開発用）

## プライバシー

入力されたプロフィール・生成結果はすべてブラウザのlocalStorageにのみ保存されます。
独自サーバーへの送信はありません（AI生成時にGoogle Gemini APIへプロンプトが送信されます）。
